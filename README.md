# 3x-ui autoinstaller (Reality + gRPC/XHTTP/WS)

Автоустановщик для 3x-ui под Ubuntu 22.04/24.04. Скрипт сам определяет систему и IP, ставит свежий 3x-ui без ручного ввода, выпускает сертификат Let's Encrypt для `<ipv4>.sslip.io`, включает HTTPS для панели через `x-ui cert`, настраивает Nginx как TLS-фронт для трёх входящих, а затем создаёт 4 VLESS inbound'а и сохраняет все данные для подключения.

## Установка

Рекомендуемый способ (работает и под root, и через sudo; меню выбора языка EN/RU остаётся интерактивным):

```bash
curl -Ls https://raw.githubusercontent.com/StraitBowyer/wqyuwymd/main/install.sh -o 3xui.sh && sudo bash 3xui.sh
```

Если вы уже под root, можно короче через процесс-подстановку:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/StraitBowyer/wqyuwymd/main/install.sh)
```

> Не запускайте `sudo bash <(curl ...)`: sudo не пробрасывает файловый дескриптор
> процесс-подстановки, и bash упадёт с ошибкой `/dev/fd/63: No such file or directory`.
> Используйте вариант со скачиванием в файл или без `sudo`, если вы уже root.

Полностью неинтерактивный запуск (язык задаётся переменной `XUI_LANG`, `ru` или `en`):

```bash
curl -Ls https://raw.githubusercontent.com/StraitBowyer/wqyuwymd/main/install.sh -o 3xui.sh && sudo XUI_LANG=ru bash 3xui.sh
```

## Требования

- Ubuntu 22.04 или 24.04
- запуск от root (или через sudo)
- белый (публичный) IPv4-адрес
- открытый порт 80 на время выпуска сертификата (HTTP-01 challenge)
- открытые порты входящих: 8443, 2087, 2083, 2096
- открытый порт панели 2053

## Поэтапно: что делает скрипт

Установка проходит в 8 шагов, каждый подписан в выводе (`[1/8] … [8/8]`):

1. **Определение системы.** Проверяет, что это Ubuntu, определяет версию, публичный IPv4/IPv6, число ядер, RAM и диск. На не-Ubuntu останавливается, на версиях кроме 22.04/24.04 предупреждает, но продолжает.
2. **Домен sslip.io.** Формирует домен вида `<ваш-ipv4>.sslip.io` — он автоматически резолвится в ваш IP, отдельный домен покупать не нужно.
3. **Зависимости.** Ставит `curl`, `nginx`, `jq`, `sqlite3`, `socat`, `openssl` и другие пакеты.
4. **3x-ui.** Ставит последнюю версию панели официальным установщиком в неинтерактивном режиме.
5. **Сертификат.** Через `acme.sh` выпускает сертификат Let's Encrypt для домена sslip.io (standalone HTTP-01, нужен свободный порт 80). Если действующий сертификат уже есть — переиспользует его, не перевыпускает.
6. **Панель по HTTPS.** Задаёт случайные логин/пароль, порт `2053`, случайный путь и включает HTTPS через `x-ui cert`. Порт подписок панели переносится с `2096` на `2095`, чтобы не конфликтовать с WebSocket-входящим.
7. **Nginx (TLS-фронт).** Поднимает Nginx, который терминирует TLS на портах `2087/2083/2096` и проксирует трафик на локальные xray-входящие (`127.0.0.1:11087/11083/11096`).
8. **Создание 4 инбаундов.** Через API панели создаёт входящие и печатает готовые ссылки `vless://…`.

## Что будет установлено

| Inbound | Порт | Транспорт | Примечание |
|---|---|---|---|
| VLESS Reality TCP | 8443 | tcp + reality | `fp=firefox`, `flow=xtls-rprx-vision`, цель — крупный сайт (Microsoft и т.п.) |
| VLESS gRPC | 2087 | grpc + TLS | за Nginx |
| VLESS XHTTP | 2083 | xhttp + TLS | за Nginx |
| VLESS WebSocket | 2096 | ws + TLS | за Nginx |

Дополнительно занятые порты: `2053` — панель, `2095` — сервер подписок 3x-ui, `11083/11087/11096` — локальные (только `127.0.0.1`) xray-входящие за Nginx.

После установки все клиентские ссылки и учётные данные панели сохраняются в:

```text
/root/3xui-install-info.txt
```

## Возможные ошибки и что делать

### `bash: /dev/fd/63: No such file or directory`
Вы запустили `sudo bash <(curl ...)`. sudo не пробрасывает дескриптор процесс-подстановки. Запустите вариант со скачиванием в файл (`curl -Ls ... -o 3xui.sh && sudo bash 3xui.sh`) или без `sudo`, если вы уже root.

### Сертификат не выпускается (шаг 5)
Причины и проверка:
- Порт 80 занят или закрыт файрволом/провайдером. Проверьте: `ss -tlnp | grep ':80 '` и правила облачного firewall/Security Group.
- Нет белого IPv4 или домен `<ip>.sslip.io` не резолвится в ваш IP. Проверьте: `curl -4 ifconfig.me` и `dig +short <ip>.sslip.io`.
- Упёрлись в лимит Let's Encrypt (частые перевыпуски). Подождите или используйте уже выпущенный сертификат — скрипт переиспользует действующий автоматически.
Логи acme.sh: `~/.acme.sh/acme.sh.log`.

### Nginx не запускается (шаг 7)
Проверьте точную причину:
```bash
nginx -t
journalctl -xeu nginx.service --no-pager | tail -30
```
Типичные случаи:
- `unknown directive "http2"` — старый Nginx (< 1.25). В актуальной версии скрипта `http2` уже задаётся в директиве `listen`, обновитесь до свежего `install.sh`.
- `bind() to 0.0.0.0:2096 failed (Address already in use)` — порт занят. Чаще всего это сервер подписок 3x-ui (по умолчанию `2096`). Актуальный скрипт переносит подписки на `2095`. Посмотреть, кто держит порт: `ss -tlnp | grep 2096`.

### Панель недоступна по HTTPS (шаг 6)
```bash
x-ui status
systemctl status x-ui --no-pager
journalctl -u x-ui --no-pager | tail -30
```
Убедитесь, что порт `2053` открыт в файрволе и что сертификат существует в `/root/cert/<домен>/`.

### xray не стартует / `invalid UUID`
Симптом в логах: `common/uuid: invalid UUID`. Это исправлено в актуальной версии скрипта (UUID клиентов брались из API в неверном формате). Обновите `install.sh` до последней версии из `main` и переустановите.

### Клиент не подключается, хотя всё «зелёное»
- Проверьте, что порт входящего открыт в облачном firewall/Security Group, а не только в `ufw`.
- Reality (`:8443`): в клиенте должны совпадать `pbk` (public key), `sid` (short id), `sni` и `fp` из ссылки.
- gRPC/XHTTP/WS: подключение идёт через Nginx с TLS, `sni` должен быть равен вашему домену `<ip>.sslip.io`.
- Быстрая проверка TLS снаружи: `openssl s_client -connect <ip>.sslip.io:2087 -servername <ip>.sslip.io` (ожидается сертификат Let's Encrypt); для `:8443` — сертификат сайта-цели Reality.

## Повторный запуск и удаление

- **Повторный запуск.** Скрипт можно запустить повторно, но он создаст ещё один комплект из 4 входящих (дубликаты). Перед повтором либо удалите старые входящие в панели, либо переустановите 3x-ui заново (`x-ui uninstall`, затем запуск скрипта). Действующий сертификат при этом переиспользуется.
- **Где посмотреть доступы позже.** Файл `/root/3xui-install-info.txt` или меню `x-ui` на сервере.
- **Полное удаление.** `x-ui uninstall` (удалит панель и xray), затем при необходимости удалите конфиг Nginx `/etc/nginx/conf.d/3xui-proxy.conf` и перезапустите Nginx.

## Кратко

Установщик рассчитан на быстрый разворачивающий запуск без ручной настройки панели и входящих. Каждый шаг подписан в выводе, а при ошибке установка останавливается и показывает причину — сверяйтесь с разделом «Возможные ошибки и что делать» выше.
